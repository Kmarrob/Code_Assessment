10.2 Problema 2: CORS - Múltiplos Valores — ✅ CORRIGIDO
Erro:

text
The 'Access-Control-Allow-Origin' header contains multiple values 
'https://cisatool.com.br,https://code-assessment-frontend.onrender.com,http://localhost:5173', 
but only one is allowed.
Causa: O CORS estava configurado com uma única string em config.CORS_ORIGIN, mas precisava aceitar múltiplas origens.

Solução Aplicada:

Arquivo backend/src/config/env.ts:

typescript
// Antes
CORS_ORIGIN: z.string().url().default('http://localhost:5173'),

// Depois
CORS_ORIGIN: z.string().default('http://localhost:5173,https://cisatool.com.br,https://code-assessment-frontend.onrender.com,https://code-assessment-898z.onrender.com'),
Arquivo backend/src/server.ts:

typescript
const corsOrigins = config.CORS_ORIGIN.split(',').map(origin => origin.trim());

app.use(cors({
  origin: function (origin: string | undefined, callback: (err: Error | null, allow?: boolean) => void) {
    if (!origin) {
      if (config.NODE_ENV === 'development') {
        return callback(null, true);
      }
      return callback(new Error('Not allowed by CORS'));
    }
    if (corsOrigins.indexOf(origin) !== -1) {
      callback(null, true);
    } else {
      callback(new Error(`Origin ${origin} not allowed by CORS`));
    }
  },
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With'],
  exposedHeaders: ['X-Total-Count', 'X-Page', 'X-Limit', 'Authorization'],
  optionsSuccessStatus: 200,
}));
Status: ✅ CORRIGIDO

Causa: O TypeScript não tinha certeza se levels[level] existia antes de incrementar.

Solução Aplicada:

typescript
private static calculateMaturityLevels(controls: any[]) {
  const levels: Record<string, number> = {
    'N/A': 0,
    '0': 0,
    '1': 0,
    '2': 0,
  };

  controls.forEach(c => {
    const level = c.maturityLevel || 'N/A';
    if (Object.hasOwn(levels, level)) {
      levels[level] = (levels[level] ?? 0) + 1;
    }
  });

  return levels;
}
